# /create-beelink-repo

Создать новый beelink-репозиторий **детерминированно**. Создание GitHub-репо
необратимо, поэтому в критическом пути НЕТ LLM — вся работа в одном bash-скрипте
`scripts/create_beelink_repo.sh`. Этот скилл — **человеческая точка входа** в ТОТ ЖЕ
движок, который дёргает демон через `create_repo`-хендлер (один движок, две точки входа).

## Args

- `name` (обязательный) — голое имя, валидируется `^[a-z0-9]([a-z0-9.-]*[a-z0-9])?$` (внутренние точки/дефисы разрешены, напр. `foo.ru`). Везде станет `beelink-<name>`.
- `vision` (опц.) — Product Vision, пойдёт в `docs/strategy/vision.md` с AUTOGENERATED-шапкой.
- `template` (опц.) — шаблон, default `init_project`.

## Шаги

НЕ реимплементируй 10 шагов. Просто вызови единый движок:

```bash
bash scripts/create_beelink_repo.sh --name "<name>" --vision "<vision>" --template "<template>"
```

Скрипт делает (строго по порядку, идемпотентно):

1. валидация имени; убедиться, что dir отсутствует и GitHub-репо нет (GET → 404);
2. копия `init_project/` (без `.env.example`);
3. регистрация в deployer (`POST /projects`);
4. реальный `.env` (gitignored) из ответа deployer + `PROJECT_NAME=beelink-<name>`;
5. `docker-compose.yml` (`container_name`, без портов наружу, `mem_limit`/`cpus`, healthcheck);
6. `docs/strategy/vision.md` (AUTOGENERATED-шапка);
7. `openspec init` + минимальный `openspec/project.md`;
8. `git init` + commit;
9. GitHub-репо (API+PAT из `$HOME/.config/gh-token`) + SSH-push (`git@github.com:Garbuzenko/...`);
10. печатает `{"url":..., "path":...}` в stdout.

## Что НЕ делать

- НЕ реимплементировать шаги в Python/inline-bash — единственный источник правды `scripts/create_beelink_repo.sh`.
- НЕ слать HTTPS-пуш (на VDS виснет) — только SSH, это уже зашито в скрипт.
- НЕ создавать `.env.example` — пишем реальный gitignored `.env`.
